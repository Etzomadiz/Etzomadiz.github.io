Project Design

Below is the design I and my client have in mind for the website I am building.

Project Name: Troop 555 website

Project Overview:

  • The Project she wants is a website for her girl scout troop.
  • This application is to inform local parents about troop 555 and the activities that they do. This website can also be used to update parents on upcoming events and activities.
  • The content on this website will have an information page about the troop and event page, a picture page and a page dedicated to the sale of girl scout cookies.

Site Map:

Page Design:

  • About:
    • The Name of the page is "About"
    • The purpose of this page is to give general information about the troop and to provide the user withcontact information and a brief history.
    • Audience: the audience for this website is either parents wanting their children to join or the kids inthe troop.
    • The content is the contact information, the leader of the troops information and history of the troop.
    • It will need HTML and CSS validation.
    • It will most likely contain hyperlinks for peoples emails.
    • There will be no buttons on the about page.
    • None.
  • Events:
    • The Name of the page is "Events"
    • The purpose of the page is to inform the users about upcoming events happening in the troop.
    • The audience is parents of kids in the troop.
    • The content of this page is information for volunteering events, information on the end of the year trip and then information on donations.
    • This page is asking users if they would like to donate so it will probably have a small optional form they can fill out.
    • The page will need css and html validation.
    • The page will have a submit and reset button for the donations form
    • User will press a button and the data will be processed theoretically.
    • Might need to figure out how to have a page written to a document to save information.
  • Gallery:
    • The Name of the page is "Gallery"
    • The purpose of the page is to show photos of the troops and their activities to make the user interested in joining the troop.
    • Audience will once again be parents and members of the troop.
    • The content of this page is a gallery of pictures that represents the experiences and events that the troop members participated in.
    • No it will not need this.
    • Will need HTML and CSS validation.
    • No buttons will be needed.
    • No actions on the page.
    • No notes regarding this page.
  • Cookies:
    • The Name of the page is "Cookies"
    • The purpose of this page is to place orders for cookies and give information about the girl scout cookies in general.
    • The audience of this page is actually anyone that wants to buy cookies from the troop.
    • Yes this page will have users input what kind of cookies they want and how many they want.
    • The data field of what they are purchasing will need to have an input and there will also need to be HTML and CSS validation.
    • The page will contain buttons and links to the cookie catalog.
    • Actions will be submitting a form.
    • No needed notes.

Dynamic Functionality on the website:

  • My client wants there to be like a scroll feature for the photos in the gallery so I will need to impliment that with javascript.
  • I will have to create a form page so that I can have user order things from the website.
  • Im also gonna be using dynamic include for the headers and footers so I need to set that up.
